Developing this full-stack FastAPI calculator was a profound learning experience that bridged the gap between backend architecture and frontend interactivity. A major highlight was implementing JWT-based authentication. Transitioning from open, unprotected endpoints to a secure, user-specific experience required careful synchronization between the FastAPI backend and the JavaScript frontend, particularly when managing the BREAD (Browse, Read, Edit, Add, Delete) lifecycle and token storage.

The most significant challenges emerged during the automated testing and CI/CD deployment phases. Configuring Playwright for End-to-End (E2E) testing exposed several UI edge cases. For instance, building the new "Edit" functionality introduced a tricky timeout bug where Playwright attempted to click a hidden "Cancel Edit" button due to loose text matching. Resolving this by enforcing exact role-based matching (exact=True) was a highly valuable lesson in writing resilient test selectors.

Furthermore, the CI/CD pipeline presented strict architectural hurdles. I encountered a severe database race condition where Pytest fixtures dropped the database tables just as the background Uvicorn server needed them for Playwright tests. Strategically splitting the pipeline running unit tests first, then spinning up a fresh server elegantly solved this. Finally, addressing Bandit security scanner warnings by migrating hardcoded keys to a git-ignored .env file reinforced critical, production level security practices. Ultimately, overcoming these automation and deployment hurdles resulted in a highly robust, containerized application ready for real-world use.
